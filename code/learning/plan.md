# 投资专业知识学习计划 (14天)

## 目标
把自己培养成专业股票分析师/基金经理水平

## 学习框架
- 第一阶段(4天): 基础投资理论
- 第二阶段(4天): 交易策略体系
- 第三阶段(3天): 技术分析补充
- 第四阶段(3天): 整合与实战

## Day1 (4/10) - 价值投资基础理论
1. 格雷厄姆《聪明的投资者》核心思想: 安全边际、市场先生
2. 银行螺丝钉投资理念: 指数基金定投、低估买入高估卖出
3. PE/PB/PS/股息率 深入理解: 各指标适用场景和局限
4. PEG指标: 彼得林奇的成长股估值法
5. 作业: 用当前持仓验证各指标是否合理

## Day2 (4/11) - 财务报表分析
1. 资产负债表: 资产质量、负债率、有息负债
2. 利润表: 营收增长、毛利率、净利率、ROE
3. 现金流量表: 经营现金流vs净利润、自由现金流
4. 杜邦分析: ROE拆解为利润率×周转率×杠杆
5. 银行螺丝钉选股标准: ROE>15%、利润稳定、估值合理
6. 作业: 分析持仓5只股票的财务报表

## Day3 (4/12) - 行业分析方法
1. 行业生命周期: 成长期/成熟期/衰退期
2. 行业竞争格局: 龙头企业、护城河分析(巴菲特)
3. 银行业分析: 净息差、不良率、资本充足率
4. 白酒行业: 高端/次高端/中端，品牌壁垒
5. 保险行业: NBV、内含价值、投资收益率
6. 作业: 对持仓行业做深度分析

## Day4 (4/13) - 基金投资理论
1. 指数基金: 跟踪误差、费率、规模
2. 主动基金: 基金经理能力评估(Alpha/Beta/夏普/最大回撤)
3. 银行螺丝钉定投法: 低估定投、正常持有、高估分批卖出
4. 股债平衡: 经典60/40组合、再平衡策略
5. 作业: 筛选适合定投的指数基金

## Day5 (4/14) - 买入策略
1. 估值买入法: PE历史百分位(10年), 低于30%分位买入
2. 分散投资: 单只不超过总资金20%, 行业不超过30%
3. 分批建仓: 三角形建仓法(3-3-4)
4. 银行螺丝钉五步选基法
5. 作业: 模拟买入策略

## Day6 (4/15) - 卖出策略
1. 估值卖出法: PE历史百分位>80%分批卖出
2. 止盈止损: 目标收益率止盈、最大回撤止损
3. 基本面恶化信号: ROE下降、营收负增长、负债飙升
4. 换股逻辑: 新机会显著优于持仓时才换
5. 作业: 制定持仓卖出条件

## Day7 (4/16) - 风险控制
1. 仓位管理: 凯利公式、固定比例法
2. 最大回撤控制: 单股止损线、组合止损线
3. 相关性分析: 持仓之间的相关性，避免同行业集中
4. 黑天鹅应对: 现金仓位的重要性
5. 作业: 设计风险控制体系

## Day8 (4/17) - 资金流向与市场情绪
1. 主力资金: 北向资金、融资融券、龙虎榜
2. 成交量分析: 量价关系、放量缩量
3. 市场情绪指标: 恐慌指数、换手率、涨停跌停比
4. 机构行为: 基金持仓变化、调研频率
5. 作业: 分析当前市场情绪

## Day9 (4/18) - 技术指标基础
1. 均线系统: MA5/10/20/60/120/250 葛兰碧八大法则
2. MACD: 金叉死叉、顶底背离、红绿柱
3. RSI: 超买超卖、背离信号
4. 作业: 对持仓股票做技术分析

## Day10 (4/19) - K线与形态
1. K线组合: 锤子线、吞没、十字星
2. 支撑压力: 前高前低、整数关口
3. 趋势线: 上升/下降/横盘趋势
4. 作业: 识别持仓股票的关键技术位

## Day11 (4/20) - 量化分析基础
1. 多因子模型: 价值因子+动量因子+质量因子+规模因子
2. 回测框架: 如何验证策略有效性
3. 夏普比率、索提诺比率、卡玛比率
4. 作业: 为当前策略做简单回测

## Day12 (4/21) - 综合投资体系搭建
1. 整合价值投资+技术分析+资金面
2. 建立股票评分模型(10+指标)
3. 建立买卖决策树
4. 作业: 输出完整的投资体系文档

## Day13 (4/22) - 投资组合管理
1. 现代投资组合理论: 有效前沿
2. 再平衡策略: 定期vs阈值触发
3. 资产配置: 股/债/现金比例
4. 作业: 设计最优投资组合

## Day14 (4/23) - 实战模拟总结
1. 回顾2周模拟交易结果
2. 分析决策中的正确和错误
3. 输出: 重新设计的专业分析引擎v6.0
4. 输出: 更新后的App设计方案
